01 — Pengantar

Apa itu
Cloud Computing?

Cloud computing adalah model pengiriman layanan komputasi melalui internet — server, penyimpanan, database, jaringan, perangkat lunak, analitik, dan kecerdasan buatan — tanpa perlu memiliki infrastruktur fisik.

01

Akses dari Mana Saja

Layanan cloud dapat diakses kapan saja melalui koneksi internet, tanpa bergantung pada perangkat atau lokasi fisik tertentu.

02

Skalabilitas Tinggi

Sumber daya komputasi ditingkatkan atau dikurangi secara otomatis sesuai kebutuhan, tanpa investasi infrastruktur baru.

03

Efisiensi Biaya

Model pay-as-you-go mengubah biaya modal besar menjadi biaya operasional fleksibel yang terhitung per jam atau per detik.

04

Keandalan Tinggi

Data di-backup di beberapa lokasi geografis berbeda, memastikan ketersediaan layanan (high availability) dan pemulihan bencana yang cepat.

05

Kolaborasi Real-time

Banyak pengguna bekerja pada dokumen atau proyek yang sama secara bersamaan dari lokasi berbeda di seluruh dunia.

06

Pembaruan Otomatis

Pembaruan perangkat lunak dan keamanan dilakukan otomatis oleh penyedia, mengurangi beban pengelolaan IT internal secara signifikan.

02 — Sejarah

Evolusi
Cloud Computing

1960-an

Mainframe & Time-sharing

Konsep berbagi sumber daya komputasi melalui terminal diperkenalkan oleh John McCarthy — cikal bakal gagasan cloud modern.

1999

SaaS Pertama — Salesforce

Salesforce merevolusi pengiriman software melalui browser, meletakkan fondasi model SaaS yang kini dipakai jutaan perusahaan.

2006

Amazon Web Services Lahir

AWS meluncurkan S3 dan EC2, menandai era cloud publik modern. Istilah "cloud computing" mulai populer di kalangan industri teknologi global.

2010-an

Era Multicloud & Container

Microsoft Azure dan Google Cloud hadir sebagai pesaing. Docker dan Kubernetes merevolusi cara deployment aplikasi berskala besar.

2020+

Cloud + AI = Masa Depan

Pandemi mempercepat adopsi cloud secara masif. Kini cloud menjadi fondasi AI generatif, edge computing, dan infrastruktur digital nasional.

03 — Model Layanan

Tiga Pilar
Cloud Computing

Masing-masing model menawarkan tingkat kontrol, fleksibilitas, dan manajemen yang berbeda — dipilih sesuai kebutuhan teknis organisasi.

Infrastructure as a Service

IaaS menyediakan infrastruktur komputasi dasar — virtual machine, jaringan, dan penyimpanan — melalui internet. Pengguna memiliki kontrol penuh atas sistem operasi dan konfigurasi server.

Cocok untuk: Tim IT yang membutuhkan fleksibilitas penuh dan sedang memigrasikan aplikasi legacy ke lingkungan cloud.

Amazon EC2 Google Compute Azure VMs DigitalOcean

Platform as a Service

PaaS menyediakan lingkungan pengembangan dan deployment yang lengkap. Developer fokus menulis kode tanpa mengkhawatirkan pengelolaan infrastruktur di bawahnya.

Cocok untuk: Developer yang ingin membangun, menguji, dan mendeploy aplikasi dengan cepat tanpa pengelolaan server.

Heroku Google App Engine Azure App Service Vercel

Software as a Service

SaaS adalah model distribusi di mana penyedia mengelola seluruh infrastruktur dan aplikasi. Pengguna cukup mengakses melalui browser tanpa instalasi apapun.

Cocok untuk: Pengguna akhir yang ingin langsung menggunakan aplikasi tanpa kerumitan teknis. Model paling umum untuk konsumen.

Google Workspace Microsoft 365 Zoom Dropbox
04 — Model Deployment

Cara Deploy
Cloud

Empat model deployment cloud dipilih berdasarkan kebutuhan keamanan, biaya, dan skala organisasi.

Public

Public Cloud

Infrastruktur dimiliki dan dioperasikan oleh penyedia pihak ketiga. Sumber daya dibagi dengan banyak pelanggan. Contoh: AWS, Azure, GCP.

Private

Private Cloud

Infrastruktur cloud eksklusif untuk satu organisasi. Dapat di-host on-premise atau oleh penyedia pihak ketiga. Keamanan lebih tinggi, biaya lebih besar.

Hybrid

Hybrid Cloud

Menggabungkan public dan private cloud, memungkinkan data dan aplikasi bergerak di antara keduanya. Fleksibilitas tinggi dengan kendali atas data sensitif.

Multi

Multi Cloud

Strategi menggunakan layanan dari beberapa penyedia cloud berbeda. Menghindari vendor lock-in dan mengoptimalkan keunggulan setiap provider.
